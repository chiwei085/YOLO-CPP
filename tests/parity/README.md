# Parity Test Skeleton

This directory is reserved for model parity checks against a Python reference
stack such as Ultralytics.

Current scope:

- `detect` parity is planned
- `classify` parity is planned
- `pose`, `segmentation`, and `obb` parity are intentionally out of scope for
  the first wave

These parity checks are not registered in CTest yet. The goal for the first
step is to keep the regular C++ test suite fast and deterministic while leaving
room for model-backed regression runners later.

Planned inputs:

- exported ONNX models in `tests/assets/models/`
- optional sample images in `tests/assets/images/`
- JSON baselines generated by a Python runner

Planned workflow:

1. Export or place ONNX assets under `tests/assets/models/`.
2. Run `tests/parity/run_parity.py` to generate reference outputs.
3. Compare C++ inference outputs against the saved references.
